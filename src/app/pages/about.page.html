About this application

You may ask how this application works, how does it calculate the single dummy results?

The single dummy analysis uses the double dummy analysis of the Bridge DDS library.

Where n is the number of samples chosen, the library generates n hands that complete the declarer and dummy hands provided, and then for each hand calculates the number of tricks declarer can take with the given strain (trump suit or no-trump).

The greater the sample size, the more accurate - but the longer it takes the calculation to run. To protect the application (and the server), the sample size is limited to 2000.

The results of the calculation are then averaged to provide an estimate of the probability of the declarer taking a certain number of tricks.

For example, if the user wants to know the probability of declarer taking 6 tricks with a given hand and dummy, the application will generate n hands that complete the declarer and dummy hands provided, and then for each hand calculate the number of tricks declarer can take with the given strain (trump suit or no-trump). The results of the calculation are then averaged to provide an estimate of the probability of the declarer taking 6 tricks.

The calculation assumes the best possible opening lead on each hand, and assumes the most successful possible play by declarer.

This obviously doesn't match real life, but it provides a useful estimate of the probability of declarer taking a certain number of tricks.

Future versions of this may try to improve on the algorithm, for example by incorporating more realistic assumptions about opening leads and play.
